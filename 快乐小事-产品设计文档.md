## 一、产品理念

### 核心命题
**在碎片化的日常中，帮助人们主动觉察和收集快乐。**

不是大事件记录器，不是日记本，是一个"快乐放大镜"——让你发现那些容易被忽略的、微小但真实的快乐瞬间。

### 设计哲学
1. **轻量先于完整**：记一件事只需要10秒，不要求完整叙事
2. **情绪先于信息**：关注"这件事让我什么感觉"而非"发生了什么"
3. **温暖先于功能**：UI的每个细节都服务于"让人感到被温柔对待"
4. **本地先于云端**：数据存在你手上，云只是锦上添花的同步工具

### 目标用户
- 忙碌的职场人，需要在高压中找到"小确幸"的出口
- 不习惯写长日记，但愿意花几秒记一笔的人
- 对美感有要求，不能接受粗糙工具的人

- CSS变量方案（`data-theme` 属性切换），按用户隔离保存
- 主题选择同步到CloudBase云端，清缓存/换设备后自动恢复

### 字体方案
- **标题/装饰**：ZCOOL XiaoWei（站酷小薇体）— 手写感、文艺气质

- Tab-bar 高度 64px，用 CSS calc 自适应各机型底部横条高度

## 三、功能模块详解

### 模块1：笑一声（哈哈机接泡泡）

**定位**：进入app的第一个tab，用游戏化方式给用户一个笑点。

- 小女孩SVG角色 + 网兜，彩色泡泡从天空落下
- 用户滑动控制小女孩左右移动接泡泡
- **金色发光 emoji 球**（固定46px，比普通球大，下落速度慢20%）接到才触发笑话
- 接到普通泡泡只有"💫 普通泡泡"短暂提示，游戏继续
- 接到 emoji 球 → 弹出随机笑话（180+条冷兔风格）→ 存入泡泡池
- 30秒超时后重置按钮，让用户主动重玩（**不再强制触发笑话**）
- 碰撞检测区域动态计算（`skyBoxH - 95`），适配不同手机高度
- 小碎步音效（Web Audio API合成，iOS每次刷新稳定解锁）

**设计思考**：放在第一个tab是因为登录后第一件事是"笑"，建立积极心理暗示。

### 模块2：记好事（快乐广场 + 回看时光 + 快乐指数）

**定位**：核心功能，日历式快乐记录 + 瀑布流浏览。

**页面结构**：
- 顶部固定区：标题 + 本周快乐指数☀️ + "记录好事"按钮 + 子tab
- 快乐广场（默认）：瀑布流双列卡片，最新在前
- 回看时光：按日期分组时间线 + 日期筛选跳转

**记录内容**：
- 文字（必填，200字以内）
- 配图（可选，自动压缩到600px宽 + 自适应质量，超800KB时云同步剥离图片）
- 快速标签（预设10种 + 茸言茸语）
- 配文（从100+古诗词中选择）
- emoji装饰（随机分配）
- 天气（自动获取深圳天气）

**本周快乐指数**：
- 右上角太阳SVG图标 + 数字
- 太阳形态随本周记录数变化（0=灰暗 → 8+=炽热金黄）
- 渐变核心 + 交替长短光线 + 柔和光晕 + 高光玻璃感
- 记得越多太阳越亮，激励持续记录

### 模块3：随Ta去（坏情绪粉碎机）

**定位**：情绪宣泄出口。快乐不是压抑负面，而是正确处理后放下。

- 纸条式输入框（手写质感）
- 5秒完整粉碎动画（纸条吸入1.5s + 碎片飘落 + 结果展示）
- 粉碎音效（白噪音+碎片敲击声3秒，Web Audio API合成）
- 随机安慰语（100+条）
- 保存粉碎记录（删除线样式展示）

### 模块4：问观音（灵签100签）

**定位**：趣味+精神慰藉。不是迷信，是"获得指引"的仪式感。

- 佛堂暗色背景 + 签筒摇动 + 竹签碰撞音效
- 100签完整解签数据（sign-data.js）
- 每签含：签级/宫位/典故/签文/诗意/解曰/仙机/精髓/完整详解/故事
- 100张真实签图GIF
- 签文历史按用户隔离保存

### 模块5：我的（个人中心）

- 圆形头像上传（居中裁剪200×200 + JPEG压缩，同步到云端）
- 自定义"快乐空间"名称（同步到云端）
- 🎨 主题切换（4色大地色系）
- 修改密码（云端+本地同步）
- 密保设置（5预设+自定义问题）
- 导出数据（JSON全量导出，只含当前用户数据）
- **🗑️ 一键清除历史数据**（新增）：清除本地三个集合数据，可选同时清除云端，有二次确认弹窗
- 退出登录（清除CloudBase认证态+内存状态重置）

### 模块6：分享快乐卡片

- 详情页底部📤分享按钮
- 使用 html2canvas 直接截取详情页DOM渲染为图片
- 输出清晰度匹配设备像素比（devicePixelRatio），无损
- 外层包裹主题背景色 + 底部水印
- 移动端 navigator.share 原生分享 / 桌面端下载PNG
- **视觉与详情页100%一致**（字体/颜色/间距/配图全部原样）

### 模块7：月度快乐报告

- 每月1号晚8点后自动弹出（2-3号兜底），每月只弹一次
- 统计上月数据：总快乐条数 / 记录天数 / 日均 / 标签TOP3 / 最走心的一条
- 按用户隔离

### 模块8：每月备份提醒

- 每月1-3号登录时弹出温馨提醒
- 可选"立即导出"（跳转到导出功能）或"下次再说"
- 每月只提醒一次

### 模块9：换设备/清缓存自动恢复

- 本地有数据：登录后展示 splash，后台静默同步云端新增数据，完成后自动刷新广场
- 本地无数据：显示「正在恢复数据...」loading 屏，从云端全量拉取，完成显示「找回 N 条快乐记录 ✨」

## 四、音效系统

所有音效使用 Web Audio API 合成（无需音频文件），确保 iOS Safari 稳定播放。

### 解锁机制（v1.3重写）
- 废弃旧的 `audioUnlocked` flag（存在刷新后不重新解锁的bug）
- 改为每次用户交互都调用 `ensureAudioRunning()`（async，真正 await resume）
- 监听 `touchstart / touchend / click / keydown` 四种事件，覆盖所有交互场景
- 所有音效函数改为 async，播放前先 await ensureAudioRunning()

### 音效列表
| 场景 | 描述 |
|------|------|
| 记录快乐 | 清脆叮咚上升音（C5→E5→G5→C6+G6闪光） |
| 粉碎情绪 | 白噪音+滤波器下扫3秒+碎片敲击 |
| 哈哈机跑动 | 短促白噪音低通（脚步声） |
| 接泡泡 | 清脆高频碰撞音 |
| 笑话弹出 | 爆裂声 |
| 摇签 | 竹签碰撞（280ms间隔循环） |
| 签落出 | 木质清脆撞击+落地弹跳 |

## 五、技术架构

### 整体架构
```
用户设备（iOS/Android/PC）
├── index.html（单文件6500+行，含全部CSS+JS+HTML）
├── sign-data.js（100签完整数据）
├── cloudbase.bundle.js（SDK 3.x自构建）
├── html2canvas.min.js（DOM截图库CDN）
├── sw.js（Service Worker离线缓存）
├── manifest.json（PWA配置）
└── signs/（100张签图GIF）

数据存储
├── IndexedDB（本地优先，_owner字段隔离多账号）
│   ├── entries（快乐记录）
│   ├── shredded（粉碎记录）
│   └── bubblePool（泡泡池）
├── localStorage（用户配置/签文历史/头像/主题）
└── CloudBase 云端（双向同步+profile存储）
    ├── entries 集合（按_username隔离）
    ├── shredded 集合（按_username隔离）
    ├── bubblePool 集合（按_username隔离，v1.3新增）
    └── profile记录（头像/空间名/主题）

托管
├── 前端：GitHub Pages（永久免费）
└── 后端：腾讯云CloudBase（19.9元/月）
```

### 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前端架构 | 单文件HTML | 简单、零构建、直接部署 |
| 数据存储 | IndexedDB + CloudBase | 离线优先 + 多设备同步 |
| 前端托管 | GitHub Pages | 免费永久、无需备案、国内可访问 |
| 登录系统 | CloudBase邮箱密码 | 云端统一认证，多设备通用 |
| 图片存储 | Base64(压缩至600px，超800KB时云同步剥离) | 简单直接，防超CloudBase 2MB限制 |
| 多账号隔离 | _owner字段(忽略大小写)+_username云端隔离 | 本地+云端双重隔离 |
| 主题系统 | CSS变量+data-theme | 零JS开销，即时切换 |
| 分享卡片 | html2canvas截取DOM | 100%还原页面视觉，无需手动模拟 |
| 离线支持 | Service Worker | 断网也能完整使用 |
| 音效系统 | Web Audio API合成 | 无需音频文件，iOS稳定可用 |

## 六、安全与数据保障

### 多账号隔离
- IndexedDB：三个 store 均有 `_owner` 字段，读取时按 `currentUsername`（忽略大小写）过滤
- 云端：所有查询按 `_username` 字段过滤，写入时携带 `_username`
- 账号切换时强制清空 `entriesCache`（防内存残留）
- 退出登录时清除 CloudBase 认证态 + 所有内存变量

### 云同步安全设计
- 同步防重入锁（`isSyncing` flag），防快速登录触发并发写入
- profile 文档（`_type: 'profile'`）在同步时自动过滤，不混入日记列表
- shredded 上传前通过 `getShredRecords()` 过滤 _owner，不上传其他用户数据
- 大图片（Base64 > 800KB）上传前自动剥离，防 CloudBase 2MB 单条限制

### 认证安全
- 退出登录调用 `cloudApp.auth().signOut()` 清除云端认证态
- 本地降级：云端不可用时可用本地缓存验证

### 数据持久性
- 前端代码：GitHub Pages永久免费
- 用户数据：CloudBase + 本地IndexedDB双份
- Profile（头像/空间名/主题）：同步到云端，清缓存后自动恢复
- 备份机制：每月提醒 + 手动JSON导出 + 一键清除（自选是否清云端）

### 最佳协作流程

### AI每次迭代后自动完成
1. 推送代码到GitHub
2. 升级sw.js版本号
3. 更新产品设计文档（MD + PDF）

### 关键经验教训
- html2canvas 是"与页面完全一致"需求的正确方案，不要手动Canvas模拟
- 多账号隔离必须在第一次写代码时就加，不要事后补
- _owner字段比较要忽略大小写
- 头像/主题等个人配置要同步到云端（localStorage会被清缓存清掉）
- Service Worker缓存顽固，每次改代码都必须升版本号
- Desktop目录有macOS沙盒保护，git add会失败；解决方案是cp仓库到WorkBuddy目录再commit+push
- iOS AudioContext刷新后必须重新 await resume，不能用flag跳过
- 碰撞检测不能用绝对像素，必须基于实际DOM高度动态计算
- bubblePool云同步容易被遗忘，三个集合要一起同步
