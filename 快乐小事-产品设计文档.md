## 一、产品理念

### 核心命题
**在碎片化的日常中，帮助人们主动觉察和收集快乐。**

不是大事件记录器，不是日记本，是一个"快乐放大镜"——让你发现那些容易被忽略的、微小但真实的快乐瞬间。

### 设计哲学
1. **轻量先于完整**：记一件事只需要10秒，不要求完整叙事
2. **情绪先于信息**：关注"这件事让我什么感觉"而非"发生了什么"
3. **温暖先于功能**：UI的每个细节都服务于"让人感到被温柔对待"
4. **本地先于云端**：数据存在你手上，云只是锦上添花的同步工具

### 目标用户
- 忙碌的职场人，需要在高压中找到"小确幸"的出口
- 不习惯写长日记，但愿意花几秒记一笔的人
- 对美感有要求，不能接受粗糙工具的人

## 三、功能模块详解

### 模块1：笑一声（哈哈机接泡泡）

**定位**：进入app的第一个tab，用游戏化方式给用户一个笑点。

- 小女孩SVG角色 + 网兜，彩色泡泡从天空落下
- 用户滑动控制小女孩左右移动接泡泡
- 接到emoji球 → 弹出随机笑话（180+条冷兔风格）→ 存入泡泡池
- 15秒超时保护
- 小碎步音效（Web Audio API合成）

**设计思考**：放在第一个tab是因为登录后第一件事是"笑"，建立积极心理暗示。

### 模块2：记好事（快乐广场 + 回看时光 + 快乐指数）

**定位**：核心功能，日历式快乐记录 + 瀑布流浏览。

**页面结构**：
- 顶部固定区：标题 + 本周快乐指数☀️ + "记录好事"按钮 + 子tab
- 快乐广场（默认）：瀑布流双列卡片，最新在前
- 回看时光：按日期分组时间线 + 日期筛选跳转

**记录内容**：
- 文字（必填，200字以内）
- 配图（可选，自动压缩到600px宽 + 自适应质量）
- 快速标签（预设10种 + 茸言茸语）
- 配文（从100+古诗词中选择）
- emoji装饰（随机分配）
- 天气（自动获取深圳天气）

**本周快乐指数**：
- 右上角太阳SVG图标 + 数字
- 太阳形态随本周记录数变化（0=灰暗 → 8+=炽热金黄）
- 渐变核心 + 交替长短光线 + 柔和光晕 + 高光玻璃感
- 记得越多太阳越亮，激励持续记录

**设计思考**：限200字强制简短降门槛；配文选古诗词提升格调感；快乐指数提供即时正反馈。

### 模块3：随Ta去（坏情绪粉碎机）

**定位**：情绪宣泄出口。快乐不是压抑负面，而是正确处理后放下。

- 纸条式输入框（手写质感）
- 5秒完整粉碎动画（纸条吸入1.5s + 碎片飘落 + 结果展示）
- 粉碎音效（白噪音+碎片敲击声3秒，Web Audio API合成）
- 随机安慰语（100+条）
- 保存粉碎记录（删除线样式展示）

**设计思考**：动画5秒是仪式感；用"随Ta去"不用"粉碎"，传达"放下"而非"对抗"。

### 模块4：问观音（灵签100签）

**定位**：趣味+精神慰藉。不是迷信，是"获得指引"的仪式感。

- 佛堂暗色背景 + 签筒摇动 + 竹签碰撞音效
- 100签完整解签数据（sign-data.js）
- 每签含：签级/宫位/典故/签文/诗意/解曰/仙机/精髓/完整详解/故事
- 100张真实签图GIF
- 签文历史按用户隔离保存

### 模块5：我的（个人中心）

- 圆形头像上传（居中裁剪200×200 + JPEG压缩，同步到云端）
- 自定义"快乐空间"名称（同步到云端）
- 🎨 主题切换（4色大地色系）
- 修改密码（云端+本地同步）
- 密保设置（5预设+自定义问题）
- 导出数据（JSON全量导出，只含当前用户数据）
- 退出登录（清除CloudBase认证态）

### 模块6：分享快乐卡片

- 详情页底部📤分享按钮
- 使用 html2canvas 直接截取详情页DOM渲染为图片
- 输出清晰度匹配设备像素比（devicePixelRatio），无损
- 外层包裹主题背景色 + 底部水印
- 移动端 navigator.share 原生分享 / 桌面端下载PNG
- **视觉与详情页100%一致**（字体/颜色/间距/配图全部原样）

### 模块7：月度快乐报告

- 每月1号晚8点后自动弹出（2-3号兜底），每月只弹一次
- 统计上月数据：总快乐条数 / 记录天数 / 日均 / 标签TOP3 / 最走心的一条
- 按用户隔离

### 模块8：每月备份提醒

- 每月1-3号登录时弹出温馨提醒
- 可选"立即导出"（跳转到导出功能）或"下次再说"
- 每月只提醒一次

## 四、技术架构

### 整体架构
```
用户设备（iOS/Android/PC）
├── index.html（单文件6200+行，含全部CSS+JS+HTML）
├── sign-data.js（100签完整数据）
├── cloudbase.bundle.js（SDK 3.x自构建）
├── html2canvas.min.js（DOM截图库CDN）
├── sw.js（Service Worker离线缓存）
├── manifest.json（PWA配置）
└── signs/（100张签图GIF）

数据存储
├── IndexedDB（本地优先）
│   ├── entries（快乐记录，_owner隔离）
│   ├── shredded（粉碎记录，_owner隔离）
│   └── bubblePool（泡泡池，_owner隔离）
├── localStorage（用户配置/签文历史/头像/主题）
└── CloudBase 云端（双向同步+profile存储）
    ├── entries 集合（按_username隔离）
    ├── shredded 集合
    └── profile记录（头像/空间名/主题）

托管
├── 前端：GitHub Pages（永久免费）
└── 后端：腾讯云CloudBase（19.9元/月）
```

### 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 前端架构 | 单文件HTML | 简单、零构建、直接部署 |
| 数据存储 | IndexedDB + CloudBase | 离线优先 + 多设备同步 |
| 前端托管 | GitHub Pages | 免费永久、无需备案、国内可访问 |
| 登录系统 | CloudBase邮箱密码 | 云端统一认证，多设备通用 |
| 图片存储 | Base64(压缩至600px) | 简单直接，不超512KB限制 |
| 多账号隔离 | _owner字段(忽略大小写) | 最小改动实现数据安全 |
| 主题系统 | CSS变量+data-theme | 零JS开销，即时切换 |
| 分享卡片 | html2canvas截取DOM | 100%还原页面视觉，无需手动模拟 |
| 离线支持 | Service Worker | 断网也能完整使用 |

## 五、安全与数据保障

### 多账号隔离
- IndexedDB：entries/_owner、shredded/_owner、bubblePool/_owner
- 云端：按_username字段查询隔离
- 匹配规则：忽略邮箱大小写（`.toLowerCase()`）
- 导出数据只含当前用户

### 认证安全
- 退出登录调用 `cloudApp.auth().signOut()` 清除云端认证态
- 本地降级：云端不可用时可用本地缓存验证

### 数据持久性
- 前端代码：GitHub Pages永久免费
- 用户数据：CloudBase + 本地IndexedDB双份
- Profile（头像/空间名/主题）：同步到云端，清缓存后自动恢复
- 备份机制：每月提醒 + 手动JSON导出

## 六、协作模式与经验

### 最佳协作流程
Doris提需求（具体+优先级）→ AI直接动手 → 推送+告知 → 真机测试 → 集中反馈 → 一次性修完 → 下一需求

### AI每次迭代后自动完成
1. 推送代码到GitHub
2. 升级sw.js版本号
3. 更新产品设计文档（MD + PDF）

### 关键经验教训
- html2canvas 是"与页面完全一致"需求的正确方案，不要手动Canvas模拟
- 多账号隔离必须在第一次写代码时就加，不要事后补
- _owner字段比较要忽略大小写
- 头像/主题等个人配置要同步到云端（localStorage会被清缓存清掉）
- Service Worker缓存顽固，每次改代码都必须升版本号
